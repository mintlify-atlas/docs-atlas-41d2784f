---

The `calculate-stats.js` script generates a `hand-crafted-stats.json` file in the repository root. This file follows the [Shields.io Endpoint Schema](https://shields.io/endpoint) format, allowing it to be consumed directly by Shields.io badges.

## File Location

The output file is written to:
```
hand-crafted-stats.json
```

This file is generated by `calculate-stats.js:80` and typically committed to the repository by the GitHub Actions workflow.

## JSON Schema

The file contains a JSON object with the following structure:

<ResponseField name="schemaVersion" type="number" required>
  The Shields.io schema version. Always set to `1`.
</ResponseField>

<ResponseField name="label" type="string" required>
  The label text displayed on the badge. Always set to `"hand crafted"`.
</ResponseField>

<ResponseField name="message" type="string" required>
  The badge message showing the percentage. Format: `"{percentage}%"`
  
  Example: `"85%"`
</ResponseField>

<ResponseField name="color" type="string" required>
  The badge color based on the human-crafted percentage. Possible values:
  - `"green"` - when percentage > 80%
  - `"yellow"` - when percentage > 50% (but ≤ 80%)
  - `"orange"` - when percentage ≤ 50%
</ResponseField>

## Example Output

### High Human Percentage (>80%)

```json
{
  "schemaVersion": 1,
  "label": "hand crafted",
  "message": "85%",
  "color": "green"
}
```

### Medium Human Percentage (51-80%)

## Source Code

The output structure is generated in `calculate-stats.js:73-78`:

```javascript calculate-stats.js
const shieldsData = {
  schemaVersion: 1,
  label: 'hand crafted',
  message: `${stats.humanPercentage}%`,
  color: stats.humanPercentage > 80 ? 'green' : stats.humanPercentage > 50 ? 'yellow' : 'orange'
};
```

## Color Logic

The badge color is determined by a ternary expression:

1. **Green** (`> 80%`): High human contribution
2. **Yellow** (`> 50%` but `≤ 80%`): Moderate human contribution  
3. **Orange** (`≤ 50%`): Low human contribution

This logic is extracted from `calculate-stats.js:77`.

## Usage with Shields.io

The JSON file is consumed by Shields.io using the endpoint feature:

```markdown
![Hand Crafted](https://img.shields.io/endpoint?url=https://raw.githubusercontent.com/USER/REPO/main/hand-crafted-stats.json)
```

Shields.io reads the JSON and renders a badge with:
- Label from `label` field
- Message from `message` field
- Color from `color` field

## Stats Object

Before generating the output file, the script calculates statistics (from `analyzeCommits` function at `calculate-stats.js:42-62`):

```javascript
const stats = {
  total: totalCount,           // Total number of commits analyzed
  ai: aiCount,                 // Number of AI-assisted commits
  human: humanCount,           // Number of human-only commits
  humanPercentage: percentage  // Rounded percentage of human commits
};
```

Only the `humanPercentage` is used in the output file's `message` field.
