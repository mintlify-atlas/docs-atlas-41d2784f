This guide will help you get the Hand Crafted Badge up and running quickly.

## Running the script

<Steps>
  <Step title="Copy the script to your repository">
    Ensure `calculate-stats.js` is in your repository root directory. See the [installation guide](/installation) for the complete script.
  </Step>

  <Step title="Run the analyzer">
    Execute the script from your repository root:

    ```bash
    node calculate-stats.js
    ```

    <Info>
    The script requires Node.js to be installed. It uses only built-in modules (`child_process` and `fs`), so no dependencies need to be installed.
    </Info>
  </Step>

  <Step title="View the output">
    The script will display statistics about your commits:

    ```bash
    --- Stats ---
    Total Commits: 42
    Hand Crafted: 38 (90%)
    AI Co-edited: 4

    Saved stats to hand-crafted-stats.json
    ```

    A `hand-crafted-stats.json` file will be created with the badge configuration:

    ```json hand-crafted-stats.json
    {
      "schemaVersion": 1,
      "label": "hand crafted",
      "message": "90%",
      "color": "green"
    }
    ```

    <Tip>
    The badge color automatically adjusts:
    - Green for &gt;80% human commits
    - Yellow for 50-80% human commits
    - Orange for &lt;50% human commits
    </Tip>
  </Step>

  <Step title="Add the badge to your README">
    Add this markdown to your `README.md`, replacing `<USER>` and `<REPO>` with your GitHub username and repository name:

    ```markdown
    ![Hand Crafted](https://img.shields.io/endpoint?url=https://raw.githubusercontent.com/<USER>/<REPO>/main/hand-crafted-stats.json)
    ```

    For example, if your repository is `digitaloracle/handcrafted`:

    ```markdown
    ![Hand Crafted](https://img.shields.io/endpoint?url=https://raw.githubusercontent.com/digitaloracle/handcrafted/main/hand-crafted-stats.json)
    ```
  </Step>
</Steps>

## Advanced usage

### Limit commits analyzed

For repositories with extensive history, you can limit the number of commits analyzed:

```bash
COMMIT_LIMIT=500 node calculate-stats.js
```

This analyzes only the last 500 commits.

### Test specific patterns

The script detects AI-assisted commits by matching against patterns in commit messages. Common patterns include:

- `Co-authored-by: GitHub Copilot <noreply@github.com>`
- `Co-authored-by: Claude <noreply@anthropic.com>`
- `Co-authored-by: cursor-ai <sos@cursor.sh>`
- `AI-Assistant:` trailers
- `Generated-by:` or `AI-Generated: true`

To see which of your commits are detected as AI-assisted, you can manually inspect your git history:

```bash
git log --format="%B%n---" --no-merges | grep -i "co-authored-by"
```

## Automation with GitHub Actions

Once you've verified the script works locally, set up the GitHub Actions workflow to automatically update the badge on every push. See the [installation guide](/installation) for workflow setup.

<Warning>
The badge won't display until `hand-crafted-stats.json` is committed and pushed to your repository's main branch.
</Warning>

## Troubleshooting

### Badge not displaying

- Ensure `hand-crafted-stats.json` exists in your main branch
- Verify the URL in your badge markdown matches your repository path
- Check that the file is accessible at the raw GitHub URL

### No commits detected

- The script requires git history to analyze. Run `git log` to verify commits exist
- If using `COMMIT_LIMIT`, ensure the limit isn't set to 0

### GitHub Action not running

- Verify the workflow file is in `.github/workflows/` directory
- Check that the workflow has `contents: write` permissions
- Ensure you're pushing to the `main` branch (or update the branch name in the workflow)
